# SPEC -- 影视森林三期优化

## 1. 目标与价值

- **解决什么问题**: 二期完成了基础打磨，但项目仍缺少核心功能（搜索增强、推荐、数据可视化等），UI 细节不够精致，用户体验有提升空间
- **为什么要做**: 从"能用"到"好用"再到"想用"，需要功能完善 + 体验优化 + 新功能扩展
- **预期收益**: 提升用户留存、增强产品竞争力、打造完整影视资源平台

## 2. 上下文与约束

- **技术栈**: SpringBoot3 + MyBatis-Plus3 + MySQL8 + Next.js 15 (App Router)
- **架构**: 用户端(client-ui/client-server) + 管理端(admin-ui/admin-server)
- **数据库**: 16 张表（content_type, movie, drama, variety, anime, short_drama, resource_online/magnet/cloud, user, user_movie_list/list_item, system_setting, resource_source, crawler_schedule/task_log）
- **部署**: NAS Docker 容器（ARM64），端口 3000/3001/8080/8081
- **约束**: 不破坏现有功能，渐进式优化

## 3. 功能需求

### A. 用户体验优化

#### A1: 首页改版 - 智能推荐
- **描述**: 当前首页是静态展示，改为动态推荐：热门内容、最新更新、猜你喜欢
- **实现**: 基于浏览量/收藏量排序推荐，后端新增推荐接口
- **优先级**: P0

#### A2: 搜索增强
- **描述**: 当前搜索仅支持标题匹配，增加：搜索建议、热门搜索、搜索历史、模糊搜索
- **实现**: 后端 LIKE 查询优化 + 前端搜索下拉建议
- **优先级**: P0

#### A3: 详情页增强
- **描述**: 增加：相关推荐（同类型/同标签）、评分分布图、分享功能
- **优先级**: P1

#### A4: 用户中心完善
- **描述**: 观看历史、收藏夹管理、个人设置（主题偏好）
- **优先级**: P1

### B. 管理端优化

#### B1: 数据统计增强
- **描述**: 增加：内容增长趋势、热门搜索词排行、爬虫成功率统计、数据质量报告
- **优先级**: P1

#### B2: 爬虫管理优化
- **描述**: 爬虫任务队列可视化、失败重试机制、数据清洗规则配置
- **优先级**: P1

#### B3: 内容审核流程
- **描述**: 新增内容审核状态（待审核/已通过/已拒绝），支持批量审核
- **优先级**: P2

### C. 性能优化

#### C1: 前端性能
- **描述**: 图片懒加载、列表虚拟滚动、SSG/ISR 优化、Bundle 分析与瘦身
- **优先级**: P1

#### C2: 后端性能
- **描述**: 热门接口缓存（Redis 或本地缓存）、分页查询优化、慢查询排查
- **优先级**: P2

### D. 新功能扩展

#### D1: 多语言支持（i18n）
- **描述**: 支持中/英文切换，为国际化做准备
- **优先级**: P2

#### D2: 暗色模式
- **描述**: 用户端和管理端支持暗色模式切换
- **优先级**: P2

#### D3: 内容标签系统
- **描述**: 为内容增加标签（标签云），支持按标签筛选
- **实现**: 新增 tag 表 + 内容关联表，前端标签展示和筛选
- **优先级**: P1

## 4. 非功能需求

- **性能**: 首页加载 < 2s，搜索响应 < 500ms
- **兼容性**: 移动端优先，支持主流浏览器
- **代码质量**: TypeScript strict，0 error，组件复用率提升
- **可维护性**: 模块清晰，API 文档完整

## 5. 验收标准

- [ ] 首页有动态推荐内容
- [ ] 搜索支持建议和历史
- [ ] 详情页有相关推荐
- [ ] 管理端数据统计更丰富
- [ ] 暗色模式可用
- [ ] 前端性能评分 > 90（Lighthouse）
- [ ] 所有功能通过测试并部署到 NAS
