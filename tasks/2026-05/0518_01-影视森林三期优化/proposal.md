# PROPOSAL -- 影视森林三期优化

## 1. 背景与动机

影视森林项目已完成两轮打磨：
- **一期**: 基础功能开发（16 张表、4 个子系统）
- **二期**: UI/UX 打磨（F1-F13 问题修复、代码审查、部署验证）

当前状态：功能可用，但缺少核心竞争力功能（推荐、搜索增强、标签系统等），UI 仍有提升空间。

## 2. 项目概述

在现有代码基础上，从"打磨"升级为"优化+扩展"：
- 用户体验升级（首页推荐、搜索增强、详情页增强）
- 管理端增强（数据统计、爬虫优化）
- 新功能扩展（暗色模式、标签系统）
- 性能优化（前端性能、后端缓存）

## 3. 技术选型

沿用现有技术栈，不引入重型依赖：
- 前端: Next.js 15 (App Router) + Tailwind CSS + shadcn/ui
- 后端: SpringBoot 3 + MyBatis-Plus 3
- 数据库: MySQL 8
- 图表: Recharts（已有）

## 4. 里程碑

| 阶段 | 目标 | 预计周期 |
|------|------|---------|
| Phase 1 | 核心体验：首页推荐 + 搜索增强 + 详情页增强 | 第 1-5 轮 |
| Phase 2 | 管理增强：数据统计 + 爬虫优化 | 第 6-9 轮 |
| Phase 3 | 体验升级：暗色模式 + 标签系统 | 第 10-13 轮 |
| Phase 4 | 性能完善：前端优化 + 用户中心 | 第 14-17 轮 |
